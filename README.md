# paizaの各スカウトのマッチ度を算出するシステム<br>
各スカウトを時間を掛けて見ているため、一時期見逃してしまうことが多々発生していた<br>
そのため、各スカウトのマッチ度を算出してマッチ度の低いスカウトは見なくても済みようなシステムを目指した<br>
<br>
ログインIDとパスワードは変更している<br>
seleniumでweb操作を自動化している<br>
paizaにログイン後にスカウト画面へ移動する<br>
普段の操作でスカウト画面に移動すると0番目のスカウトが強制選択状態になっているため見ていなくても既読状態になってしまう<br>
そのため、一部既読状態のスカウトがあっても本当に既読したのかが判断できない
対策としてこのプログラムではスカウト一覧を取得後に一番下の未読スカウトから0番目までのスカウトを対象にして取得している<br>
取得する内容は各スカウトに添付してある求人のURLを取得する<br>
これを取得後に構文解析が可能なAIに条件と一緒にURLを渡すことでマッチ度を算出させる<br>
AIへ渡す例文<br>
ーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーー<br>
以下の条件と求人内容を比較して点数を出してほしい<br>
・条件<br>
客先常駐である場合-1点<br>
リモートワークが週2日以上の場合+1点<br>
年間休日数が120日未満の場合-1点<br>
フレックスタイムがある場合+1点<br>
・求人ページ<br>
https://paiza.jp/career/job_offers/9999999<br>
ーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーーー<br>
AIからの回答の点数によって求人と自分自身のマッチ度を算出する<br>
また、AIが必要な理由として「客先常駐」という文字列があるだけでは判断できないという問題がある<br>
「客先常駐という文字列があったら-1点」としてしまうと「客先常駐ではありません」という文章でマイナス評価になってしまうためである<br>
この場合はむしろプラス評価になるはずだが、ただの検索では対応ができないため構文解析ができるAIが必要である<br>
